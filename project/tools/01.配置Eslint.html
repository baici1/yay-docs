    # 配置Eslint

    # 前言

    其实我之前也看了很多文章，配置eslint。总是配置失败，然后有很奇怪的错误，今天就花了几个小时吧这个eslint全局配置一下。

    因为现在项目并不是团队合作，所以我一般就直接全局配置，方便使用。

    eslint官网：https://eslint.org/open in new window

    # 步骤

    1. 下载eslint

    进入setting.json文件里面，进行简单的配置

        // 启用Eslint
  "eslint.enable": true,
  "eslint.options": {
    "configFile": "填写全局eslint配置文件的路径",
    //指定vscode的eslint所处理的文件的后缀
    "extensions": [
      ".js",
      ".vue",
      ".ts",
      ".tsx"
    ]
  },
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  },
    1. 全局安装eslint

    命令：

    npm i eslint -g

    然后进入eslint的包内，创建.eslintrc.js文件

    1. 写入相关配置
